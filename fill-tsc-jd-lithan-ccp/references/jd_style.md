# JD line style — Column F "Job responsibilities (aligned with JD)"

How to write each one-line JD entry so it matches the existing Lithan CCP TSC sheets.

## Rules

1. **One sentence, one line.** No bullet sub-points, no line breaks. Roughly 15–30 words.
2. **Start with an action verb** — Develop, Build, Design, Manage, Implement, Deploy,
   Gather, Monitor, Evaluate, Engage, Configure, Maintain, Plan…
3. **Align with the Objective (Column D).** The JD line is the same competency
   restated as *what this person does on the job* — not a copy of the generic
   framework objective. Convert the framework definition into a concrete duty.
4. **Contextualise to the job role and the organisation.** Reference the actual
   systems/products the role works on (e.g. for Tertiary Infotech: TMS, automation
   tools, courseware automation/generation, SSG API integration, data & AI). Don't
   leave the line generic if the reference sheet is specific.
5. **Stay consistent with sibling rows.** Same voice, tense, and level of specificity
   across all rows of the sheet. Mirror the reference sheet's phrasing patterns.
6. **Don't invent tools** — tools belong in Column G, not the JD line. Keep F about
   the responsibility.
7. **Fill only rows where Column E = "Yes"** unless told otherwise.

## Worked examples

These are real pairs (Objective → JD line) from filled sheets in this client's workbook.

**Software Engineering Manager** (reference sheet "3.3 Software Eng Manager"):

| Skill | Objective (D, abbreviated) | JD line (F) |
|-------|----------------------------|-------------|
| Applications Development | Build and maintain applications, reusing and integrating components… | Build and maintain TMS and automation applications, reusing and integrating components to meet evolving functional and analytical needs. |
| Applications Integration | Integrate data or functions from one application with another… | Develop and implement integration plans to connect TMS with finance systems and external APIs using appropriate middleware. |
| Change Tracking / Configuration Tracking | Track and manage changes and revisions in software projects… | Manage version control and change tracking for TMS, automation tools, and integrated systems to ensure integrity and compliance. |
| Project Management | Plan, organise, monitor and control all aspects of an IT programme… | Manage end-to-end delivery of TMS and automation programmes, ensuring resource alignment, timeline adherence, and achievement of business goals. |

**AI/ML Engineer** (sheet "2.4 AI ML Engineer"):

| Skill | Objective (D, abbreviated) | JD line (F) |
|-------|----------------------------|-------------|
| Business Needs Analysis | Identify and scope business requirements and priorities… | Gather and scope business requirements to define AI/ML solution scope for TMS and courseware automation initiatives. |
| Computational Modelling | Develop, select and apply algorithms… to enable systems to learn, adapt, predict | Develop and apply algorithms and computational methods that enable models to learn, adapt and make predictions from data. |
| Data Engineering | Develop processes to collect, store, extract, transform, load and integrate data | Build and maintain data pipelines to collect, transform and integrate data for AI/ML model development. |
| Text Analytics and Processing | Identify, extract and analyse text data to discover themes, patterns, trends | Build text analytics and NLP solutions to extract themes, patterns and trends from text data. |

## Pattern to copy

`<Action verb> <the competency, concretely> <for / to support> <the role's systems &
business context>, <optional outcome/qualifier>.`

Keep the outcome clause only when it adds meaning (e.g. "…to ensure integrity and
compliance", "…to meet business goals"). Drop it when it would just pad the line.
